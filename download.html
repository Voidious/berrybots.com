Installing: Raspberry Pi (Raspbian "wheezy")

Download and unzip the files where ever you like, e.g.:

cd ~
unzip berrybots_rpi_1.1.0.zip

This will create a berrybots subdirectory containing the binaries, font, and sample ships and stages. You can view usage instructions by running berrybots.sh with no arguments:

./berrybots.sh
Usage:
  berrybots.sh [-nodisplay] <stage.lua> <bot1.lua> [<bot2.lua> ...]
  OR
  berrybots.sh -packstage <stage.lua> <version>
  OR
  berrybots.sh -packbot <bot.lua> <version>

Sample ships are in the bots/sample subdirectory and sample stages are in the stages/sample subdirectory. You run a match by passing a stage then any number of ships. For example, to run a jousting match:

./berrybots.sh stages/sample/joust.lua bots/sample/chaser.lua \ bots/sample/jouster.lua

To abort a match, press ctrl+C.

Installing: Mac / Linux / Windows

On these platforms, BerryBots is run through a graphical interface (GUI). Launching the app is slightly different on each platform, but otherwise it functions almost identically on all three.

  • Mac OS X: Drop the .app into your Applications folder and run it. The first time, you'll need to select a base directory for BerryBots. This is where ships and stages will be located, and the API docs, sample ships, and sample stages will be copied into subdirectories. BerryBots supports Mac OS X 10.7 (Lion) or later.
  • Linux / Windows: Unzip where ever you like and run berrybots.sh (Linux) or BerryBots.exe (Windows). Ships and stages are loaded from the bots and stages subdirectories. API documentation is in the apidoc subdirectory.

Compiling from source: Raspberry Pi (Raspbian "wheezy")

Compiling BerryBots on the Raspberry Pi is simple. Check out or unzip the code where ever you like, change into that directory, and run the build script.

cd ~
unzip berrybots_src_1.1.0.zip
cd berrybots
make rpi

This will take 10-15 minutes to complete.

Compiling from source: Mac / Linux / Windows

To compile BerryBots on these platforms, you'll need g++ (or another C++ compiler). On Mac OS X, you can get it from the XCode developer tools, which is on the OS install CD and in the Mac App Store. On most Linux systems, you probably already have it. On Windows, you can use MinGW.

The build process on each platform is slightly different, but the high level steps are the same.

  • Download SFML 2.0 libraries or compile them from source.
  • Download wxWidgets 2.9.4 libraries or compile them from source.
  • Compile libarchive from source, linking with zlib.
  • Download the BerryBots source code and compile it, linking against the above 3 libraries. A modified version of LuaJIT is part of the BerryBots source tree and is compiled/linked by the BerryBots build process
