Installing binaries

First, if you're on a Mac, you may need to install X11. On Mountain Lion (10.8), you need to grab it from the XQuartz project. On previous versions of OS X, it can be installed from the developer tools.

Download and unzip the files where ever you'd like, eg:

cd ~
unzip berrybots_rpi_1.0.0.zip

This will create a "berrybots" subdirectory containing the binaries, font, and sample stages and ships. You can view usage instructions by running berrybots.sh with no arguments:

./berrybots.sh
Usage:
  berrybots.sh [-nodisplay] <stage.lua> <bot1.lua> [<bot2.lua> ...]
  OR
  berrybots.sh [-nosrc] -packstage <stage.lua> <version>
  OR
  berrybots.sh [-nosrc] -packbot <bot.lua> <version>

You run a match by passing a stage then any number of ships, with the optional -nodisplay to run at max speed with no graphics.

Package a stage into a .tar.gz file with -packstage so you can easily share it. Similarly, package a ship or team with -packbot. The -nosrc parameter will compile your source into Lua bytecode and package that instead, if you prefer not to share your source code.

Sample stages are in the "stages" subdirectory and sample ships are in the "bots" subdirectory. For example, to run a jousting match:

./berrybots.sh stages/sample.joust.lua bots/sample.chaser.lua \ bots/sample.jouster.lua

To abort a match, just close the window (Mac/Linux) or press CTRL+C at the terminal.

Compiling from source

To compile BerryBots, you'll need g++ (or another C++ compiler). On most Linux systems, you probably already have it. On a Mac, you may need to install XCode developer tools. Again, unzip the download where ever you'd like and cd to that directory.

cd ~
unzip berrybots_src_1.0.0.zip
cd berrybots

On Mac or Linux, you also need the SFML 2.0 shared libraries. This is the cross-platform graphics library used by BerryBots. You can get them from sfml-dev.org, from the binary downloads above, or compile them from source. Put them in berrybots/sfml-lib.

Next, look in the "scripts" subdirectory for the appropriate shell script to compile BerryBots with g++ on your platform: ./scripts/build_rpi.sh on Raspberry Pi, ./scripts/build_osx.sh on Mac, or ./scripts/build_linux.sh on Linux. This will take 10-15 minutes on the Raspberry Pi. It should be pretty quick (under a minute) on a modern Mac or Linux machine.

I have yet to create a nice build system like with "make" or similar to save you some of the platform-specific build commands and tie in the SFML stuff. I promise to get things a bit more organized soon.
