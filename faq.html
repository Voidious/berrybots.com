What is BerryBots?

BerryBots is a programming game designed for the Raspberry Pi, and also available on other platforms (currently Mac and Linux). It tries hard to cater to beginner programmers while retaining a lot of depth for seasoned software developers. You can start by trying to navigate your ship through a simple maze, fight through custom stages with a wide range of gameplay styles, work your way up to battling against other people's ships, or design your own gameplay by programming stages.

How do I write a bot?

Ships are coded in Lua, a lightweight and flexible programming language designed for just this type of application, where one program is embedded into another. Take a look at some of the sample bots, the BerryBots API, or Lua.org to get started.

What can a ship do?

A ship can...

  • ...move by firing its thrusters.
  • ...fire lasers at enemy ships - lasers are fast, weak, and fly until they hit a wall or a ship.
  • ...fire torpedos - torpedos are strong, slow, move a specific distance, move through walls and ships, and then blow up, doing damage and knockback to all ships within the blast radius.
  • ...see other ships and certain gameplay events in their field of vision (no walls in the way).
  • ...in a team game, the ship program controls all ships on the team from a single program, with all ships' combined vision and no need for communication between ships to coordinate strategy or share information.

What can a stage do?

A stage can...

  • ...configure the stage size, walls, zones, starting points, team size, and battle mode, and pre-load stage ships.
  • ...monitor everything occurring in the game, decide when a round ends, when the match is over, and who wins.
  • ...modify almost everything about ships, including destroying ships, reviving ships, moving ships, modifying a ship's speed, heading, or energy, and disabling or enabling a ship's thrusters, laser gun, or torpedo gun.
  • ...send custom events to ships to coordinate stage ships or communicate gameplay events to player ships.
  • ...draw text on the screen, such as a scoreboard, number of extra lives, congrats when a ship scores a point, or a taunt when a ship is destroyed.

This looks an awful lot like Robocode / Robot Battle / RoboWar / Logo... ?

I'm a huge fan of Robocode. I've probably spent more time writing Robocode bots than anyone ever. Robocode was inspired by Robot Battle, which drew inspiration from earlier tank battle games... So BerryBots was certainly influenced by Robocode and all of its predecessors. I think BerryBots brings some cool new stuff to the table and I hope you'll agree. (And by all means, check out Robocode and all those other games, too!)

What makes BerryBots special?

First, it's designed from the ground up to run on the Raspberry Pi. You don't need anything besides what comes in Raspbian to get started writing and running your bots with a terminal and a text editor. The game engine and the Lua programming language are pretty fast and the 2D graphics are GPU accelerated with OpenVG. You should have no problem running battles with 5-8 moderately complex bots at a smooth 60 frames per second.

I've gone to great lengths to make BerryBots especially welcoming to beginners. Everything about the ship API and game rules is designed to be as simple and intuitive as possible. I hope that folks teaching intro programming classes, especially on the Raspberry Pi, will consider checking out BerryBots. And it should scale smoothly to novice, moderate and advanced programmers, competing with each other in battle mode, race tracks, custom stages, or designing gameplay on their own stages.

Perhaps the most unique aspect of BerryBots as compared to other programming games is the programmable stage. The stage is a user written program loaded at run time, just as ships are. In the stage program, you can configure walls, zones, starting points, and pre-loaded ships designed for whatever gameplay you have in mind for the stage. The stage gets a chance to run code each tick, just as each bot does. It can monitor the game, move ships around, change their speed or energy, destroy and revive ships, send custom events to ships, write text to the screen, decide who wins and when the round or match is over. The sample stages include mazes, a race track, jousting, several arcade inspired single player games, as well as battle stages designed for 1v1, free-for-all, or team battles. And these only scratch the surface of what could be done with the stage API.

Is it safe to run other people's bots?

Yes. Lua code has no network access. In BerryBots, the Lua I/O library is disabled, so user programs don't have general file system access either.

Ships and stages can still load and run Lua code from other source files on disk, but only from below the directory of the Lua program loaded by BerryBots. So if you load a stage from /berrybots/stages/awesome.stage.lua, it can only load other Lua source files from below /berrybots/stages. When you run a packaged bot or stage, it's extracted to its own directory under berrybots/cache and can only access other files that came packaged with it.

Can't you make a nonviolent programming game?

I gave some serious thought to avoiding the idea of having bots shoot at each other in any way. I decided that the gameplay dynamics of it are just too interesting and familiar to gamers to avoid it entitely. That said, I have a couple points to offer:

  • I've been careful to keep the violence squarely in the "cartoon violence" realm. The game uses simple 2D vector graphics. There are no detailed explosions or anything photorealistic. Lasers and torpedos are not typical real world weapons. Ships don't "die", they are "destroyed" (and can be revived!).
  • A stage has to enable battle mode to allow ships to shoot at each other and a lot can be done with battle mode off. Many of the sample stages do not use battle mode - their gameplay is entirely based on ships moving around the stage. If you're averse to spaceships shooting each other, limit yourself (or your students / children) to these types of stages.

What version of Lua is BerryBots using?

BerryBots uses a modified LuaJIT 2.0.0, which itself is a highly optimized version of Lua 5.1 with some features of 5.2. The changes made for BerryBots are centered on security and packaging of ships and stages, specifically:

  • Disable the I/O library.
  • Only allow programs to load source code ("require", "loadfile", "dofile") from paths below the directory of the primary program loaded by BerryBots (i.e., your ship or stage).
  • Paths passed to "loadfile" and "dofile" are relative to the calling file, not the top level program file. (The package path for "require" is still absolute.)
  • Lua tracks all files opened. This allows BerryBots to load a ship or stage and crawl all source files referenced, so it can package all necessary files into a single zip.
