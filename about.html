Overview

BerryBots is a programming game. A player programs a ship that moves around the stage, sees gameplay events, and shoots at other ships. The APIs and game rules are simple and it's easy to get started in just a few minutes.

The stage is also configured and controlled by a program, so the gameplay on every stage is different. The sample stages include mazes, jousting, a race track, several classic arcade style games, and a few battle stages. These only scratch the surface of what can be done with the stage API, but you could stay busy for quite a while just writing ships for the sample stages.

Background

BerryBots tries to follow in the footsteps of Robocode, Robot Battle, Logo, and many other great programming games. You "play" these games by writing code to control a bot that moves around the screen. They're often used in educational settings because they're a great way to make learning to code a fun and interesting experience.

BerryBots' origins can be traced back to a discussion at the RoboWiki, a plea from a legendary Robocoder to take Robocode and package it nicely for the Raspberry Pi, or perhaps create a new lightweight version of it. At the time, it sounded like a daunting task. A few months later, I got Raspberry Pi fever. A few folks tried running Robocode on the Rpi and saw pretty bad performance. I ordered one to try writing a new programming game with the Raspberry Pi as the target platform.

I began with the goal of creating a simplified, Robocode-esque game with good performance on the Raspberry Pi. What I ended up with is a lightweight, flexible game that runs plenty fast on the Raspberry Pi, has simpler APIs and game rules than Robocode, but also introduces some gameplay elements that add a lot of variety and gameplay depth, like walls, programmable stages, simpler team support, and a secondary weapon.

Open source

BerryBots source code is released under the zlib license. Grab it from the Download page or the GitHub repository.

Credits

  • Lua and LuaJIT: The programming language used to write ships and stages.
  • SFML: A cross-platform OpenGL graphics library used for the main BerryBots game graphics on Mac / Linux / Windows.
  • wxWidgets: A cross-platform widgets library used for the full GUI version of BerryBots on Mac / Linux / Windows.
  • ajstarks' OpenVG library for Raspberry Pi: The graphics library I used for the BerryBots prototype. BerryBots still uses some of it, mainly the font support and OpenGL init/cleanup code on the Raspberry Pi.
  • libarchive and zlib: Support for packaging bots as tarred / gzipped (.tar.gz) files in the GUI version.
  • Raspberry Pi: The tiny awesome computer that inspired BerryBots.
  • Robocode: BerryBots is far from a Robocode clone, but I've spent the last 7 years addicted to this amazing programming game - BerryBots is certainly influenced by Robocode and would never have existed without it.
