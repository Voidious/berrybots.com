Overview

BerryBots is a programming game. A player programs a ship that moves around the stage, sees gameplay events, and shoots at other ships. The API and game rules are simple and it's easy to get started in just a few minutes.

The stage is also configured and controlled by a program, so the gameplay on every stage is different. The sample stages include mazes, jousting, a race track, several arcade style games, and a few battle stages. You can program custom stages for tournaments or to focus on certain types of programming problems, like pathfinding or classification algorithms.

Background

BerryBots tries to follow in the footsteps of Robocode, Robot Battle, Logo, and many other great programming games. You "play" these games by writing code to control a bot that moves around the screen. They're often used in educational settings because they make learning to code a fun and interesting experience.

BerryBots' origins can be traced back to a discussion at the RoboWiki, a request from a legendary Robocoder to take Robocode and package it for the Raspberry Pi, or perhaps create a new, lightweight version. It sounded like a daunting task. A few months later, I got Raspberry Pi fever. People tried running Robocode on the Pi and saw pretty bad performance. I ordered one to try writing a new programming game with the Raspberry Pi as the target platform.

I began with the goal of creating a simplified, Robocode-esque game with good performance on the Raspberry Pi. What I ended up with is a lightweight, flexible game that runs fast enough for the Raspberry Pi, has simpler APIs and game rules than Robocode, but also introduces a few elements that add a lot of gameplay depth, like walls on the playing field, programmable stages, simpler team support, and a secondary weapon.

Open source

BerryBots source code is released under the zlib license. Grab it from the Download page or the GitHub repository.

Credits

  • Raspberry Pi: The tiny awesome computer that inspired BerryBots.
  • Lua and LuaJIT: The programming language used to write BerryBots ships and stages.
  • SFML: A cross-platform OpenGL graphics library used for the game graphics on Mac, Linux, and Windows.
  • wxWidgets: A cross-platform widgets library used for the GUI on Mac, Linux, and Windows.
  • ajstarks' OpenVG library for Raspberry Pi: The graphics library I used for the BerryBots prototype. Some of it is still used on the Raspberry Pi - mainly the font support and OpenGL init/cleanup code.
  • libarchive and zlib: Support for packaging bots as tarred / gzipped (.tar.gz) files.
  • KineticJS: An HTML5 Canvas Javascript framework used for the BerryBots replay graphics.
  • Robocode: BerryBots is far from a Robocode clone, but I've spent the last 7 years addicted to this amazing programming game... BerryBots was definitely influenced by Robocode and would never have existed without it.
